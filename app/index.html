Start Guide

1. Go to https://string-db.org/ → Search → Organisms → Search for your organism or add an organism if not available.

2. Click ‘Go to download page’ → Download one of the Interaction Data files (I recommend ...protein.physical.links.detailed...) and one or more of the Accessory Data files (I recommend downloading all except the last three). Move these files to your desired location.

3. Open Interaction Data files and one or more of the Accessory Data files in this app.

4. Click ‘Build Network’.

App basics:
The app displays network graphs of nodes (representing proteins) and links/edges (representing connections between proteins, such as physical interactions). The nodes and links have data associated with them in the form of variables. You can change the size or colour of the nodes and links based on these variables. You can also visualise the variables as graphs (hover over “Full Network” at the bottom to see the view menu).

Shortcuts and Hotkeys:
• Spacebar → Toggle physics
• F → Toggle between Full Network and previous view
• S → Toggle between Selected Nodes view and previous view
• Z → Zoom to selected nodes
• A or Ctrl+A → Select all
• I → Invert selection
• + → increases selection to include nodes connected to the currently selected nodes
• - → decreases selection (undoing the selection increase)
• B → Toggle Brush Select
• L → Toggle Lasso Select
• Right click → Cancel operation (e.g., turns off brush select if brush select is active)
• M → Add (“move”) selection to a collection
• G → Toggles grab nodes mode (allows you to click and drag nodes to move them)
• Backspace → With your mouse hovered over a slider, value input box, or colour selector, backspace resets it to the default value.
• Backspace → With your mouse hovered over the main part of the screen (the canvas), backspace goes back to the previous view (e.g., from a graph view back to the Full Network view)

Using custom files (advanced)
Custom files can be used in this app. For custom link files, the first two columns should list pairs of nodes that are linked/connected. The other columns can contain data on any variables associated with the links.

For custom node info files, the first column should have the node ID (matching the node IDs in the links file), and the other columns can contain data on any variables associated with the nodes. Files for variables with a hierarchical nature should have one column heading containing the word “parent” and another column heading containing the word “child” with the rows of the columns containing the variable category names. Upload a separate file assigning each network node to a category (i.e., assigning each network node to a mind map node).

The app accepts CSV, TSV, Pipe-separated, semicolon-separated, Space-separated, and .h5 files.

About StringScape

Beta version 1.0

The StringScape network visualiser app was developed for viewing network data from the STRING database, but can be used to view any type of network data.

This app was developed by Joel Tregurtha as part of a PhD program at the University of Canterbury, New Zealand. PhD funded by The MacDiarmid Institute.

The code was generated using AI models, including Gemini, Codex, and Claude.

More information on this app can be found here: StringScape Home

This is an open-source project. The source code is available here: https://github.com/JoelTre/StringScape/tree/main
