Guide: Using AI to make apps for science

By Joel Tregurtha (creator of StringScape)

AI can be a very helpful tool to accelerate scientific research. However, AI is well known for making mistakes. Using AI to write code mitigates this issue. This is because you can review the code to check it does exactly what it should. In other words, rather than asking AI to do something, you ask AI to write code that does that thing. You can then check the code before using it for your research. You can do this even if you have never worked with computer code before.

Below are my recommendations for how best to use AI to write code for research apps.

Simple apps

If making a simple app, you can ask any chat-bot AI (Gemini, ChatGPT, Claude, etc.) to make you an HTML app and describe what you want the app to do. For example:

“Make me a modern, dark-themed HTML app where I can upload one or more files. The app displays a scatter plot of protein_size vs protein_weight with data from different uploaded files coloured differently on the plot. Include buttons to do these: download the scatter plot, set x or y axis to logarithmic, toggle a grid overlay, and download a CSV with the uploaded files merged into one and proteins under 10 kDa removed.”

In your message to the AI, include details about the format of the file. For example:

“The uploaded files will be PDB or mmCIF.”

Or

“The uploaded files are CSVs in this format: [paste the first few rows of an example CSV so the AI can see the column headings and cell data format].”

The AI will then give you the code for the app. Copy and paste this into a text/code editor (like Notepad or VS Code) and save it as an HTML file (e.g., my_research_app.html). Double click on the HTML file in file explorer to open it in a browser. You can then use and test the app.

If you want to make changes to the app (something is not working or you want to add a feature), just describe to the AI what change you want it to make. It will give you new code that you can copy and paste into the html file. Save the file and refresh the tab to see the new app version.

Complex apps

For more complex apps, it is better to use AI in a coding app like Visual Studio Code. Download VS Code (it's free!), open your draft app (the .html file) in VS Code, and ask the AI agent on the right hand side to make any changes you want.

You can send the AI agent in VS Code 50 free messages per month. If you need to send more than this and are a student, you can apply for GitHub Education. It will take a few days to get accepted and then you will have a much higher usage limit. You can also buy more credits if needed.

Popular alternatives to VS Code for AI-assisted coding include Codex, Windsurf, Replit, Cursor, and Antigravity.

Test your app and review the code!

Don’t trust that the AI has done everything correctly, because it probably hasn’t! Before using the app for proper research or deploying it online, test the app to make sure everything works as it should. If the app processes data, upload a test file and check to see if the output is exactly what it should be.

Read the code to check it is doing what it should. If you don’t know how to read computer code, copy and paste any section of the code (or all the code) into any AI and ask it to explain in simple English what the code is doing. You can also ask AI to add comments within the code that explain what each bit does.

Mitigate security issues

Before you deploy your app to the web, you should mitigate security issues. Here are some key things to do (you can ask AI for help if unsure):

1) Separate your code

Move all the JavaScript into a separate JavaScript file (called script.js), stored in the same folder as your HTML file. The JavaScript is the code between the <script> and </script> tags in the HTML file. In your HTML file, where the JavaScript used to be, have this instead:

<script id="main-script" src="script.js"></script>
2) Remove inline events

Replace onclick attributes with event handlers. For example, change:

<button id="guide-btn" onclick="openStartGuide()">Open Guide</button>

To:

<button id="guide-btn">Open Guide</button>

And add this to your script.js:

document.getElementById('guide-btn').addEventListener('click', openStartGuide);
3) Add a Content-Security-Policy (CSP)

Add a CSP meta tag to the <head> of your HTML file. A CSP tells the browser exactly which domains, resources, and script origins are trusted. Steps 1 and 2 above allow you to enforce a much stricter policy by completely banning inline scripts, heavily mitigating Cross-Site Scripting (XSS) injection attacks.

4) Use Subresource Integrity

If your app imports external libraries (like D3.js), use a site like srihash.org to generate a SHA-384 hash. Paste the hash into your code like this:

<script src="https://d3js.org/d3.v7.min.js" integrity="sha384-CjloA8y00+1SDAUkjs099PVfnY2KmDC2BZnws9kh8D/lX1s46w6EPhpXdqMfjK6i" crossorigin="anonymous"></script>

This acts as a digital fingerprint, protecting your users against supply chain attacks if the external hosting server is ever compromised.

5) Sanitize user, file, and AI inputs

External data—such as file names, text typed into search boxes, or strings generated by AI agents—is untrusted input. If an app renders these strings using innerHTML, a maliciously named file or text payload could inject harmful scripts into your page.

For simple UI text changes, force the browser to treat input strictly as a string by using textContent: 

element.textContent = file.name;

When you need to safely insert dynamic data into complex HTML layouts, use a dedicated escaping function to instantly make any code characters inert: 

function escapeHtml(value) {
    return String(value)
        .replace(/&/g, '&amp;')
        .replace(/</g, '&lt;')
        .replace(/>/g, '&gt;')
        .replace(/"/g, '&quot;')
        .replace(/'/g, '&#39;');
}

Deploy to the web

If you want to share your app with others, you can either just share the HTML and JavaScript files or you can deploy it on the web. If deploying it on the web, I would recommend using GitHub Pages. It’s super easy and free. Create a GitHub account and upload the app files to a new GitHub repository. The upload files need to be named index.html and script.js.

At the top of the repository page click Settings, then click Pages on the left panel, and deploy your app as a page. It will then say “Your site is live at [website URL]”. You also have the option to set a custom URL if you own one.
